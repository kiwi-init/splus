Temp log file leaks when open() throws
extensions/g-cloud/src/services/logs-service/LogsView.tsx : 1431–1454
concern

The IDE-export path writes a log file to tmpdir() and only registers the 60 s unlinkSync via setTimeout after await open(filePath, preferredIDE) resolves. If the IDE bundle is missing, open() rejects, cleanup is never scheduled, and the file is left behind permanently. The catch block only shows a toast.
